# Before Backend Edit Checklist

- [ ] I understand the requested behavior.
- [ ] I inspected relevant existing code.
- [ ] I inspected relevant tests.
- [ ] I identified repository conventions.
- [ ] I identified internal libraries and wrappers from usage.
- [ ] I identified validation commands from evidence or marked them unknown.
- [ ] I identified affected files/modules.
- [ ] I identified risks and approval gates.
- [ ] I have a minimal implementation plan.
- [ ] I know which tests to add/update.

## Stop Before Editing If

- [ ] Requirements are ambiguous.
- [ ] Design is missing for non-trivial work.
- [ ] Auth/security impact is unclear.
- [ ] Migration is needed but unapproved.
- [ ] Production config is affected.
- [ ] CI/CD or infrastructure is affected.
- [ ] Internal library behavior is unknown and critical.
- [ ] The requested change requires broad rewrite.
