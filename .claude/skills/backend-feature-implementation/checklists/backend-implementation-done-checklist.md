# Backend Implementation Done Checklist

- [ ] Required behavior implemented.
- [ ] Edge cases handled.
- [ ] Error handling added or preserved.

- [ ] Idempotency considered where relevant.

## Code

- [ ] Changes are small and focused.
- [ ] Existing style preserved.
- [ ] Existing abstractions reused.
- [ ] No unrelated formatting.
- [ ] No invented internal APIs.
- [ ] No unsafe generated-code edits.

## Tests and Validation

- [ ] Tests added/updated.
- [ ] Relevant validation command discovered.
- [ ] Validation run or explicitly not run.
- [ ] Failures analyzed.
- [ ] Residual risks documented.

## Backend Concerns

- [ ] Data consistency checked.
- [ ] Security/trust checked.
- [ ] Integration failure handling checked.
- [ ] Performance impact checked.
- [ ] Observability impact checked.
- [ ] Operations impact checked.

## Summary

- [ ] Files changed listed.
- [ ] Behavior summarized.
- [ ] Validation summarized.
- [ ] Follow-ups listed.
