# Backend Exit Criteria

## Functional Criteria

- [ ] Behavior matches design or user request.
- [ ] Edge cases are handled.
- [ ] Error cases are handled.
- [ ] Backward compatibility is preserved or explicitly changed.

## Code Criteria

- [ ] Changes are small and reviewable.
- [ ] Existing style is preserved.
- [ ] Internal libraries are used according to repo evidence.
- [ ] No broad unrelated rewrite.
- [ ] No generated code edited manually unless approved.
- [ ] No secrets introduced.

## Backend Risk Criteria

- [ ] Authorization impact checked.
- [ ] Data consistency checked.
- [ ] Idempotency checked where relevant.
- [ ] Integration failure handling checked.
- [ ] Observability checked.
- [ ] Performance implications checked.
- [ ] Operational impact checked.

## Validation Criteria

- [ ] Tests added or updated.
- [ ] Build/compile/typecheck run or documented.
- [ ] Unit tests run or documented.
- [ ] Integration tests run or documented.
- [ ] Lint/static analysis run or documented.
- [ ] Failures fixed or reported.
- [ ] Residual risks listed.

## Final Summary Criteria

- [ ] Files changed listed.
- [ ] Behavior changed summarized.
- [ ] Validation results included.
- [ ] Tests not run explained.
- [ ] Follow-ups included.
