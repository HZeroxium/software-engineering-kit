# Before Final Answer Checklist

Use before the final response.

## Work Summary

- [ ] I can explain what changed.
- [ ] I can explain why it changed.
- [ ] I can list files touched.
- [ ] I can separate confirmed facts from assumptions.

## Validation

- [ ] Relevant tests were run, or reason for not running is documented.
- [ ] Build/typecheck/lint/static analysis was run when available and relevant, or reason for not running is documented.
- [ ] Failures were inspected and addressed or escalated.
- [ ] I did not invent commands or test results.

## Safety

- [ ] No secrets or sensitive data were exposed.
- [ ] No unrelated broad rewrite was performed.
- [ ] No unapproved dependency upgrade was performed.
- [ ] No unapproved migration, production config, CI/CD, infrastructure, auth/security, or destructive change was performed.

## Quality

- [ ] Diff is minimal and reviewable.
- [ ] Existing style is preserved.
- [ ] Edge cases are mentioned.
- [ ] Residual risks are mentioned.
- [ ] Next steps are useful and not generic.
