# Application Architecture Checklist

- [ ] Use case is clear.
- [ ] Command/query distinction is clear.
- [ ] Orchestration owner is clear.
- [ ] Business logic is not only in transport layer.
- [ ] Domain logic is not hidden in infrastructure.
- [ ] Transaction boundary is explicit.
- [ ] Side effects are ordered safely.
- [ ] Failure handling is defined.
- [ ] Dependencies are explicit.
- [ ] Existing repo conventions are followed.
- [ ] Internal library behavior is confirmed or marked unknown.
- [ ] Tests cover orchestration and failure cases.
