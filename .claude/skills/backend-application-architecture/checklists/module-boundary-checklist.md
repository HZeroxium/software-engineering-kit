# Module Boundary Checklist

- [ ] Module responsibility is clear.
- [ ] Public interface is explicit.
- [ ] Internal implementation is hidden.
- [ ] Dependency direction is acceptable.
- [ ] No circular dependencies.
- [ ] Shared code has clear ownership.
- [ ] Domain module does not depend on infrastructure unexpectedly.
- [ ] Tests can target module behavior.
- [ ] Module naming matches business capability where relevant.
- [ ] Refactor plan is minimal and reviewable.
