# Event-Driven Checklist

- [ ] Event is named as a business fact.
- [ ] Producer identified.
- [ ] Consumers identified or unknown consumers considered.
- [ ] Schema versioning considered.
- [ ] Required and optional fields clear.
- [ ] Sensitive data minimized.
- [ ] Ordering assumptions documented.
- [ ] Duplicate delivery handled.
- [ ] Dead-letter or poison-message behavior defined.
- [ ] Replay/recovery considered.
- [ ] Contract tests planned.
