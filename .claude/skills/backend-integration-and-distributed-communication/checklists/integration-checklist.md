# Integration Checklist

- [ ] Caller and callee identified.
- [ ] Communication style selected intentionally.
- [ ] Contract/schema owner identified.
- [ ] Timeout/deadline defined.
- [ ] Retry policy defined.
- [ ] Idempotency analyzed.
- [ ] Duplicate handling defined.
- [ ] Partial failures identified.
- [ ] Fallback or compensation considered.
- [ ] Observability plan exists.
- [ ] Tests cover failure modes.
- [ ] Internal library behavior confirmed or marked unknown.
