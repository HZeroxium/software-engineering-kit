# Backend Before-Merge Checklist

## Must Pass or Be Explicitly Waived

- [ ] Blocking findings resolved.
- [ ] Public contract changes reviewed.
- [ ] Auth/security changes reviewed.
- [ ] Data/migration changes reviewed.
- [ ] Integration failure modes reviewed.
- [ ] Tests added/updated.
- [ ] Validation commands run or documented.
- [ ] Observability impact reviewed.
- [ ] Rollback path understood.
- [ ] Residual risks documented.

## Strongly Recommended

- [ ] Important findings resolved or tracked.
- [ ] Runbook updated if operational behavior changes.
- [ ] API docs updated if contract changes.
- [ ] Config examples updated if config changes.
- [ ] Metrics/logs/traces verified if production path changes.
