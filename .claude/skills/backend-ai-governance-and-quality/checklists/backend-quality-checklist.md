# Backend Quality Checklist

- [ ] Behavior and expected outcome are clear.
- [ ] Critical invariants are identified.
- [ ] Edge cases are covered.
- [ ] Failure modes are covered.
- [ ] Unit/integration/contract tests considered.
- [ ] Security/privacy tests considered.
- [ ] Observability validation considered.
- [ ] Manual review needed is stated.
- [ ] Build/typecheck/lint/static analysis identified if repo evidence exists.
- [ ] Residual risks are documented.
