# Applied AI Backend Checklist

- [ ] AI use case is justified.
- [ ] Deterministic alternative considered.
- [ ] Model/provider boundary identified.
- [ ] Input data sensitivity reviewed.
- [ ] Prompt/context construction reviewed.
- [ ] Retrieval design reviewed if RAG is used.
- [ ] Output schema and validation defined.
- [ ] Tool/function permissions defined.
- [ ] Human approval boundaries defined.
- [ ] Fallback behavior defined.
- [ ] Cost/latency budget considered.
- [ ] Evaluation strategy defined.
- [ ] Monitoring strategy defined.
