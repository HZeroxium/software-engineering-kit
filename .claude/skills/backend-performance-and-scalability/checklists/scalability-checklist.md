# Scalability Checklist

- [ ] Growth dimension identified.
- [ ] Expected traffic/data growth estimated or marked unknown.
- [ ] Stateless/stateful bottlenecks identified.
- [ ] Database scalability reviewed.
- [ ] Queue/job scalability reviewed.
- [ ] External provider quota reviewed.
- [ ] Tenant/noisy-neighbor risk reviewed.
- [ ] Horizontal scaling assumptions checked.
- [ ] Capacity metric identified.
- [ ] Scaling trigger identified.
