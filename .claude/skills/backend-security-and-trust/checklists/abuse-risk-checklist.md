# Abuse Risk Checklist

- [ ] Public/anonymous access checked.
- [ ] Expensive operations bounded.
- [ ] Rate limit or quota considered.
- [ ] Payload size limit considered.
- [ ] Pagination limits considered.
- [ ] Replay protection considered.
- [ ] Idempotency considered.
- [ ] Retry amplification considered.
- [ ] Queue/job overload considered.
- [ ] AI/provider cost abuse considered where relevant.
- [ ] Monitoring for abuse considered.
