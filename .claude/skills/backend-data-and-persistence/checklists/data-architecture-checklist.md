# Data Architecture Checklist

- [ ] Data owner identified.
- [ ] Source of truth identified.
- [ ] Readers and writers identified.
- [ ] Schema/data model fits access patterns.
- [ ] Constraints enforce critical invariants.
- [ ] Transaction boundary identified.
- [ ] Consistency requirement clear.
- [ ] Query patterns identified.
- [ ] Index needs checked.
- [ ] Cache impact checked.
- [ ] Migration impact checked.
- [ ] Retention/deletion/privacy checked.
- [ ] Multi-tenancy checked.
