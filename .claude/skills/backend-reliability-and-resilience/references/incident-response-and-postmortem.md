# Incident Response and Postmortem

## Incident Readiness

- Detect user impact.
- Triage cause.
- Mitigate quickly.
- Communicate status.
- Preserve evidence.
- Follow up with corrective actions.

## Runbook Should Include

- Symptoms.
- Dashboards.
- Logs/traces to inspect.
- Common causes.
- Mitigation steps.
- Rollback steps.
- Escalation contacts.
- Safety warnings.

## Postmortem Should Cover

- User impact.
- Timeline.
- Root causes and contributing factors.
- Detection gaps.
- Mitigation gaps.
- Action items with owners.
