# API Design Checklist

- [ ] API/interface style selected intentionally.
- [ ] Caller and use case clear.
- [ ] Request fields clear.
- [ ] Response fields clear.
- [ ] Required vs optional fields clear.
- [ ] Validation behavior defined.
- [ ] Error model defined.
- [ ] Retryability defined.
- [ ] Idempotency considered.
- [ ] Pagination considered for list APIs.
- [ ] Filtering and sorting constrained.
- [ ] Versioning and compatibility checked.
- [ ] Sensitive fields excluded or protected.
- [ ] Authorization implications checked.
- [ ] Contract tests or examples planned.
