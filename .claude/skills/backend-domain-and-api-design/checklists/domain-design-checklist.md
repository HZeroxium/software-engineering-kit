# Domain Design Checklist

- [ ] Business capability identified.
- [ ] Actors identified.
- [ ] Core domain concepts identified.
- [ ] Entity vs value object distinction clear.
- [ ] Aggregate or consistency boundary identified where relevant.
- [ ] Business invariants listed.
- [ ] State transitions listed.
- [ ] Invalid transitions listed.
- [ ] Domain events identified where relevant.
- [ ] Ownership boundary identified.
- [ ] Domain logic not placed only in transport layer.
- [ ] Business terms match user/product language.
- [ ] Assumptions and unknowns listed.
